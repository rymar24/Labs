================================    Microsoft Windows networks(SAMBA) =======================================
Зайти пользователем user - нажать Alt+F1 и ввести логин и пароль user

Установить клиент smbclient (sudo apt install smbclient)
Отобразить ресурсы, предоставленные в общий доступ на windows-машине (sudo smbclient -L 10.7.0.251)
Создать каталог share (sudo mkdir /mnt/share)
Установить поддержку файловой структуры Windows  ( sudo apt-get install cifs-utils )

смонтировать smb-ресурс, предоставленный на windows-машине, в локальную файловую систему
(sudo mount -t cifs //10.7.0.251/Shara /mnt/share)
Проверить смонтированный каталог - (sudo cd /mnt/share) (ls -l)

================ Client -  Microsoft Windows networks(SAMBA) ======================

Установить пакет samba (apt install samba)
Установить пакет samba-client (apt install smbclient)
Запустить mc (sudo mc) 
найти файл /etc/samba/smb.conf и открыть в редакторе (sudo mc и нажать F4)
Добавить в файл /etc/samba/smb.conf секция global строки :
	client min protocol = SMB2
	client max protocol = SMB3
	
Перегрузить samba - (sudo service smbd restart)	
Установить smbclient (sudo apt install smbclient)
Отобразить ресурсы, предоставленные в общий доступ на windows-машине (smbclient -L 10.7.180.115)
Создать каталог /mnt/hdd1 (sudo mkdir /mnt/hdd1)
Смонтировать smb-ресурс, предоставленный на windows-машине, в локальную файловую систему
		mount -o vers=3.0,username=guest //10.7.180.154/shara /mnt/hdd1
запустить mc (sudo mc) - зайти в катало /mnt/hdd1 и проверит наличие файлов		

================================ Сервер FTP  =======================================
Установить ftp-сервер и проверить работу (sudo apt install vsftpd)
Открыть файл /etc/vsftpd.conf
Разрешить пользователям осуществлять запись в директорию, 
убрав символ # комментария для опции write_enable:

Перезапустить сервер (sudo service vsftpd restart)
Запустить mc, нажать F9, выбрать -
	
========== Server -  Microsoft Windows networks(SAMBA) =============================
	sudo apt  install samba samba-common system-config-samba
	
Создаем новый файл конфигурации /etc/samba/smb.conf со следующим содержимым	
 
[global]
workgroup = WORKGROUP
server string = Samba Server %v
#netbios name = ubuntu1604
security = user
map to guest = bad user
name resolve order = bcast host
dns proxy = no
bind interfaces only = yes

[Public]
   path = /srv/public
   writable = yes
   guest ok = yes
   guest only = yes
   read only = no
   create mode = 0777
   directory mode = 0777
   force user = nobody
   
 Создаем директорию для хранилища и устанавливаем ей необходимые права
sudo mkdir -p /srv/public
sudo chown -R nobody:nogroup /srv/public
sudo chmod -R 0775 /srv/public

Перезапускаем сервис
sudo service smbd restart
