
=======================================    Сеть (LAN и WiFi) ===============================================
Показать все сетевые интерфейсы (sudo ifconfig)
Показать конфигурацию сетевого интерфейса enp0s3 (sudo ifconfig enp0s3)
Активировать (поднять) интерфейс eth0 (sudo ifup enp0s3)
Деактивировать (опустить) интерфейс eth0 (sudo ifdown enp0s3)
Выставить интерфейсу eth0 ip-адрес и маску подсети (sudo ifconfig enp0s3 10.0.0.250 netmask 255.255.0.0)
Перевести интерфейс eth0 в promiscuous-режим для «отлова» пакетов (sniffing) (ifconfig enp0s3 promisc)
Отключить promiscuous-режим на интерфейсе eth0 (sudo ifconfig enp0s3 -promisc)
Активировать интерфейс eth0 в dhcp-режиме (sudo dhclient enp0s3)
Вывести локальную таблицу маршрутизации (sudo route -n)
Задать ip-адрес шлюза по умолчанию - default gateway (sudo route add -net 0/0 gw 10.7.0.101)

Отобразить имя компьютера (sudo hostname)
Отображает статистику интерфеса enp0s3 с выводом такой информации(sudo ethtool enp0s3)
Отображает все установленные сетевые соединения (sudo netstat -tupn)
Отображает все сетевые соединения по протоколам TCP и UDP (sudo netstat -tupln)
Отобразить весь трафик на TCP-порт 80 (обычно - HTTP) (sudo tcpdump tcp port 80)


============================================ IPTABLES (firewall) ============================================

iptables — утилита командной строки, является стандартным интерфейсом управления работой межсетевого экрана (брандмауэра) 

Отобразить все цепочки правил (iptables -t filter -nL)

Удалить все пользовательские цепочки правил в filter-таблице (iptables -t filter -X)
Запретить пинговать узел  - (iptables -t filter -A INPUT -p icmp --icmp-type echo-request -j REJECT)
Проверить с Windows (ping IP) - где IP - адресс вашего линукса
Разрешить пинговать - (iptables -t filter -A INPUT -p icmp --icmp-type echo-request -j ACCEPT)
Снова Проверить с Windows (ping IP)
================================   Остановка системы =============================================

Остановить систему (sudo shutdown -h now )
sudo shutdown -h hours:minutes & 	запланировать остановку системы на указанное время (sudo shutdown -h 15:30 )
Отменить запланированную по расписанию остановку системы (sudo shutdown -c)
Перегрузить систему (sudo shutdown -r now)
Выйти из системы (sudo logout)

================  Создание и Форматирование файловых систем ========================================

Создайте в VirtualBox еще один диск  Disk2
Разбить Disk2 на 2 разделы (ext3 fat32)  (sudo fdisk /dev/sdb) потом последовательно нажимать  n, t(83,b), w
Отформатировать 1 раздел в ext3  ( sudo mkfs /dev/sdb1 )
Отформатировать 2 раздел в fat32 ( sudo mkfs -t vfat /dev/sdb2 )
Создать каталоги /mnt/hdd1 /mnt/hdd2 ( sudo mkdir /mnt/hdd1 /mnt/hdd2 )
Смонтировать раздел 1 в директорий /mnt/hdd1 ( sudo mount /dev/sdb1 /mnt/hdd1 )
Смонтировать раздел 2 в директорий ( sudo mount -t vfat /dev/sdb1 /mnt/hdd2 ) 

Скопировать файлы в смонтированные каталоги 
		(sudo cp /var/log/syslog /mnt/hdd1/file1.txt)
		(sudo cp /var/log/syslog /mnt/hdd2/file2.txt)
Oтобразить информацию о смонтированных разделах с отображением общего, доступного и используемого пространства (sudo df -h)
Подсчитать и вывести размер, занимаемый директорией /var (sudo du -f /var)
Вывести размер и имена файлов и директорий, с соритровкой по размеру (sudo du -sk * | sort -rn)
Проверить раздел sda1 на наличие bad-блоков (sudo badblocks -v /dev/sda1)
Проверить/восстановить целостность linux-файловой системы раздела sda1 (sudo fsck /dev/hda1 )
Проверить/восстановить целостность файловой системы fat раздела sdb2 (sudo fsck.vfat /dev/sdb2)

===========================================================   Архивация ============================================
Создать каталог ~/cat/arch (mkdir ~/cat/arch)
Скопировать в него файлы с каталога /var/log 

Создать tar-архив с именем log.tar содержащий файлы с каталога /var/log  (sudo tar cvf log.tar /var/log)
Распаковать log.tar  в домашний каталог  (tar xvf log.tar)
Создать архив log.tar.gz с сжатием Gzip (tar czvf log.tar.gz /var/log/*)
Распаковать log.tar с Gzip (gunzip log.tar.gz)
Создать архив log.tar с сжатием Bzip2  (tar -cjvf log.tar.bz /var/log*)
Распаковать log.tar с Bzip2 (bunzip2 log.tar.bz)
Создать файл file2.txt (cat Hello World > file2.txt)
Сжать file2.txt и переименовать в file2.gz (gzip file2.txt)
Разжать file2.gz в file2.txt (gunzip file2.gz file2.tzt)
Показать содержимое архива
Создать архив и сжать его с помощью bzip2 (bzip2 log.bz /var/log/*)
Разжать архив и распаковать его (bunzip2 /var/log/*)

убить» процесс с PID 98989 «на смерть» (без соблюдения целостности данных) (sudo kill -9 98989)
